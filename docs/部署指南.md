# HotFeed 部署指南

> 目标域名：`prenceyours.xyz` | 方案：Cloudflare Pages（前端）+ Render（后端）

---

## 架构说明

```
用户手机/电脑浏览器
    │
    ├──► prenceyours.xyz (Cloudflare Pages)
    │    └── 前端 Vue3 静态文件
    │         │
    │         └── fetch API ──► hotfeed-backend.onrender.com (Render)
    │                              └── FastAPI + SQLite + APScheduler
    │                                   ├── 每10分钟抓取18个平台
    │                                   ├── jieba去重引擎
    │                                   ├── 7天数据缓存
    │                                   └── 历史Top15保存3天
    │
    └── 手机/电脑均可访问，无需开电脑
```

## 第一步：推送代码到 GitHub

### 1.1 在 GitHub 创建仓库

1. 打开 https://github.com/new
2. Repository name: `hotfeed`
3. 选择 **Private**（私有）
4. **不要**勾选 "Add a README"
5. 点击 **Create repository**

### 1.2 在本地推送代码

```bash
cd /workspace/HotFeed聚合热点

# 添加远程仓库（替换 S-PREFENCE 为你的 GitHub 用户名）
git remote add origin https://github.com/S-PREFENCE/hotfeed.git

# 推送
git push -u origin main
```

> 如果提示需要认证，使用 GitHub Personal Access Token（Settings → Developer settings → Personal access tokens → Generate new token，勾选 `repo` 权限）。

---

## 第二步：部署后端到 Render

### 2.1 注册 Render

1. 打开 https://render.com
2. 点击 **Get Started** → 用 GitHub 账号登录
3. 授权 Render 访问你的 GitHub

### 2.2 创建 Web Service

1. 进入 Dashboard → **New +** → **Web Service**
2. 选择刚推送的 `hotfeed` 仓库
3. 填写配置：

| 项目 | 值 |
|------|-----|
| **Name** | `hotfeed-backend` |
| **Region** | Singapore（离国内最近的免费区域） |
| **Runtime** | Python 3 |
| **Build Command** | `pip install -r src/backend/requirements.txt` |
| **Start Command** | `cd src/backend && python -m uvicorn main:app --host 0.0.0.0 --port $PORT` |
| **Plan** | Free |

4. 点击 **Create Web Service**
5. 等待构建完成（约 2-3 分钟）
6. 记下分配的域名，格式类似：`https://hotfeed-backend.onrender.com`

### 2.3 验证后端

访问 `https://hotfeed-backend.onrender.com/api/v1/health`，应返回：
```json
{"status": "ok", "timestamp": "..."}
```

> ⚠️ Render 免费层会在 15 分钟无请求后休眠，首次访问可能需要 30-60 秒唤醒。之后每 10 分钟的定时抓取会自动保活。

---

## 第三步：部署前端到 Cloudflare Pages

### 3.1 前置准备

确保你的域名 `prenceyours.xyz` 已在 Cloudflare 管理（DNS 托管在 Cloudflare）。

### 3.2 创建 Pages 项目

1. 登录 Cloudflare Dashboard → **Workers & Pages**
2. 点击 **Create application** → **Pages** → **Connect to Git**
3. 选择 GitHub → 授权 → 选择 `hotfeed` 仓库
4. 填写构建配置：

| 项目 | 值 |
|------|-----|
| **Project name** | `hotfeed` |
| **Production branch** | `main` |
| **Framework preset** | None |
| **Build command** | `cd src/frontend && npm install && npm run build` |
| **Build output directory** | `src/frontend/dist` |

5. 在 **Environment variables** 中添加：

| 变量名 | 值 |
|--------|-----|
| `VITE_API_BASE` | `https://hotfeed-backend.onrender.com/api` |

> ⚠️ 替换为你在第二步获得的 Render 域名

6. 点击 **Save and Deploy**
7. 等待构建完成（约 1-2 分钟）

### 3.3 绑定自定义域名

1. 进入 Pages 项目 → **Custom domains** → **Set up a custom domain**
2. 输入 `prenceyours.xyz`
3. Cloudflare 会自动配置 DNS（如果域名已在 Cloudflare 管理）
4. 等待 SSL 证书生效（通常几分钟内）

## 常见问题

### Q: Render 免费层休眠怎么办？
A: 后端每 10 分钟自动抓取数据，这个定时任务会保持服务活跃。如果仍休眠，可以在 Cloudflare Workers 设置一个 cron job 每 5 分钟 ping 一次 `https://hotfeed-backend.onrender.com/api/v1/health`。

### Q: 数据库会丢失吗？
A: Render 免费层的文件系统是临时的，每次部署/重启会清空。SQLite 数据会丢失。解决方案：
- 短期：可接受，因为每 10 分钟会重新抓取
- 长期：迁移到 Render PostgreSQL（免费 90 天）或 Supabase（永久免费 500MB）

### Q: 更新代码后怎么部署？
A: 只需 `git push origin main`，Cloudflare Pages 和 Render 都会自动重新部署。

### Q: BBC/CNN/YouTube/Reddit 抓取失败？
A: 这些国际源的 SSL/TLS 在某些服务器上可能被限制。不影响国内平台和 Google News 的数据。可在后续迭代中通过代理解决。
