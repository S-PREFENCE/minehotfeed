# HotFeed 部署指南 — Cloudflare 全家桶

> **方案**：Cloudflare Workers（后端）+ D1（数据库）+ Cron Triggers（定时抓取）+ Pages（前端）
> **目标域名**：`prenceyours.xyz`
> **总成本**：**¥0**（Cloudflare 免费额度完全够用）

---

## 架构说明

```
用户浏览器
    │
    └──► prenceyours.xyz (Cloudflare Pages)
         │   前端 Vue3 静态文件（GitHub Pages / Pages 均可）
         │
         └── fetch /api/v1/hotspots
             │
             ▼
         hotfeed-backend.<你的账户>.workers.dev (Cloudflare Workers)
              │
              ├── D1 SQLite (5GB免费, 数据持久化)
              │     ├── dedup_hotspots (当前热点)
              │     ├── history_hotspots (历史记录)
              │     └── fetch_status (抓取状态)
              │
              ├── Cron Trigger */10 * * * *
              │     └── 每10分钟自动抓取18个平台 → 去重 → 存D1
              │
              └── 数据源
                    ├── 国内11平台 (UAPIS API): 微博/抖音/百度/知乎/头条/B站等
                    ├── RSS (rss2json代理): 人民日报/Google News/BBC/CNN
                    └── Reddit JSON API
```

### 为什么选这个方案？

| 对比项 | Cloudflare | Vercel | Render | 腾讯云Base |
|--------|-----------|--------|--------|-----------|
| **价格** | 完全免费 | 免费(有限制) | 需信用卡$1预授权 | 按量付费 |

## 第一步：注册 Cloudflare 账号

1. 打开 https://dash.cloudflare.com/sign-up
2. 注册账号（邮箱即可，无需信用卡）
3. 登录 Dashboard

## 第二步：创建 D1 数据库

### 2.1 创建数据库实例

1. 左侧菜单 → **Workers 和 Pages**
2. 点击 **D1 SQL 数据库** 标签页
3. 点击 **创建数据库**
4. 名称填：`hotfeed-db`
5. 位置选择：**APAC（亚太地区）**——延迟更低
6. 点击 **创建**

### 2.2 初始化表结构

1. 在数据库详情页，点击 **控制台** 标签
2. 把以下 SQL 粘贴到编辑框并执行：

```sql
CREATE TABLE IF NOT EXISTS dedup_hotspots (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    subtitle TEXT DEFAULT '',
    unified_heat REAL DEFAULT 0,
    category TEXT DEFAULT '社会',
    trend TEXT DEFAULT 'stable',
    platform TEXT DEFAULT '',
    url TEXT DEFAULT '',
    region TEXT NOT NULL DEFAULT 'domestic',
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS history_hotspots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    rank INTEGER NOT NULL,
    title TEXT NOT NULL,
    subtitle TEXT DEFAULT '',
    unified_heat REAL DEFAULT 0,
    category TEXT DEFAULT '社会',
    platform TEXT DEFAULT '',
    url TEXT DEFAULT '',
    region TEXT NOT NULL DEFAULT 'domestic',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS fetch_status (
    key TEXT PRIMARY KEY,
    value TEXT
);

CREATE INDEX IF NOT EXISTS idx_dedup_region ON dedup_hotspots(region);
CREATE INDEX IF NOT EXISTS idx_dedup_category ON dedup_hotspots(category);
CREATE INDEX IF NOT EXISTS idx_dedup_updated ON dedup_hotspots(updated_at);
CREATE INDEX IF NOT EXISTS idx_history_date ON history_hotspots(date);
```

3. 执行成功后会显示 "Success"

### 2.3 复制 Database ID

在数据库详情页顶部，找到 **Database ID**，类似：

```
xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

**复制这个 ID**，下一步要用。

## 第三步：部署 Worker 后端

### 3.1 方式A：通过 wrangler CLI 部署（推荐）

#### 安装 Node.js + wrangler

你的电脑需要安装 Node.js 18+，然后：

```bash
npm install -g wrangler
```

#### 登录 Cloudflare

```bash
wrangler login
```

会自动打开浏览器，授权登录。

#### 配置 database_id

编辑项目中的 `worker/wrangler.toml`，把 `<你的D1数据库ID>` 替换为第二步复制的真实 ID：

```toml
[[d1_databases]]
binding = "DB"
database_name = "hotfeed-db"
database_id = "这里粘贴你的真实Database ID"  # ← 改这行
```

#### 本地测试

```bash
cd worker
npm install
npx wrangler dev
```

访问 `http://localhost:8787/api/v1/health` 测试。

#### 部署到 Cloudflare

```bash
cd worker
npx wrangler deploy
```

部署成功后会输出类似：
```
Published hotfeed-backend (生产环境)
  https://hotfeed-backend.<你的账户>.workers.dev
```

**记下这个 URL**，后面配置前端时需要。

### 3.2 方式B：通过 Dashboard 直接部署（无需命令行）

如果不想装 Node.js：

1. 进入 **Workers 和 Pages** → **创建应用程序** → **创建 Worker**
2. 名称：`hotfeed-backend`
3. 把 `worker/src/index.ts` 的内容粘贴进去
4. 但这种方式不支持 D1 绑定和 Cron，**强烈建议用方式A**

## 第四步：验证 Worker 后端

部署完成后，在浏览器或用 curl 测试：

```bash
# 健康检查
curl https://hotfeed-backend.<你的账户>.workers.dev/api/v1/health
# 预期: {"code":0,"data":{"status":"ok","timestamp":"..."}}

# 手动触发一次数据抓取
curl -X POST https://hotfeed-backend.<你的账户>.workers.dev/api/v1/refresh
# 预期: {"code":0,"data":{"message":"抓取任务已触发"}}

# 查询热点数据
curl "https://hotfeed-backend.<你的账户>.workers.dev/api/v1/hotspots?region=all&limit=10"
# 预期: 返回 domestic 和 international 数组
```

> ⚠️ 首次 `/api/v1/refresh` 可能需要 30-60 秒（要逐个抓取18个平台），请耐心等待。

## 第五步：部署前端

### 方案A：Cloudflare Pages（推荐，与 Worker 同域）

#### 5A.1 创建 Pages 项目

1. Dashboard → **Workers 和 Pages** → **创建应用程序** → **Pages** → **连接到 Git**
2. 选择 GitHub 授权 → 选择 `minehotfeed` 仓库
3. 配置构建设置：

| 项目 | 值 |
|------|-----|
| **项目名称** | `hotfeed-frontend` |
| **生产分支** | `main` |
| **框架预设** | None |
| **构建命令** | `cd src/frontend && npm install && npm run build` |
| **构建输出目录** | `src/frontend/dist` |

4. 点击 **保存并部署**

#### 5A.2 绑定自定义域名

1. 进入 Pages 项目 → **自定义域** → **设置自定义域**
2. 输入 `prenceyours.xyz`（确保域名 DNS 已托管在 Cloudflare）
3. 自动配置 DNS + SSL 证书

#### 5A.3 配置路由（关键！）

为了让前端的 `/api/*` 请求转发到 Worker，需要在 Pages 项目中添加路由规则：

1. Pages 项目 → **设置** → **函数** → **路由匹配**
2. 或者在仓库根目录创建 `_routes.toml`（已有此文件）：

```toml
# _routes.toml - 让 /api 请求走 Worker，其余走 Pages 静态文件
[[routes]]
pattern = "/api/*"
zone_name = "prenceyours.xyz"

[[routes]]
pattern = "prenceyours.xyz/*"
```

> ⚠️ 如果不想折腾路由，更简单的办法是直接在前端代码里写死 Worker URL（见方案B）。

### 方案B：GitHub Pages + 前端直连 Worker（最简单）

如果你已经开启了 GitHub Pages（gh-pages 分支已推送），只需修改前端 API 地址指向 Worker：

编辑 `src/frontend/src/composables/useHotspots.js` 第6行：

```js
// 改为你的 Worker 实际地址
const API_BASE = 'https://hotfeed-backend.<你的账户>.workers.dev/api'
```

然后重新构建推送：

```bash
cd src/frontend
npm run build

# 更新 gh-pages 分支（脚本方式或手动复制 dist 内容）
```

GitHub Pages 地址：`https://s-prefence.github.io/minehotfeed/`

## 常见问题

### Q: 抓取失败、没有数据？
A: 访问 Worker 的 `/api/v1/refresh` 手动触发一次。查看 Worker 日志：
```bash
wrangler tail
```

### Q: Cron 没有触发？
A: 确认 `wrangler.toml` 中 `crons = ["*/10 * * * *"]` 已正确配置。Dashboard → Workers → hotfeed-backend → 触发器 可以看到 Cron 状态。

### Q: 国内平台抓取失败？
A: UAPI 是第三方免费服务，偶尔不稳定。Reddit/BBC/CNN 通过 RSS/JSON API 抓取，相对稳定。

### Q: 如何更新代码？
A: 修改代码后 `git push origin main`，然后：
- 前端：Pages 自动重新部署
- 后端：`cd worker && npx wrangler deploy`

### Q: D1 数据库满了怎么办？
A: 免费额度 5GB，对于文本型热点数据来说基本用不完。如果担心，可以在 `saveDedup()` 中加清理逻辑，只保留最近7天数据。

### Q: 想换回 FastAPI 版本？
A: `src/backend/ 目录保留了完整的 Python FastAPI 代码，随时可以切换回去部署到任意支持 Python 的服务器。
