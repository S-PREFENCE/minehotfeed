# HotFeed 部署操作手册 — Cloudflare 全家桶

> **看完就能做，不需要任何前置知识。**
> **全程零费用，不需要信用卡，不需要备案。**

---

## 0. 准备工作：你需要什么

| 物品 | 说明 |
|------|------|
| 一台能上网的电脑 | Windows / Mac 都行 |
| 一个邮箱 | QQ邮箱、Gmail 都行，用来注册 Cloudflare |
| 一个 GitHub 账号 | 已经有 `S-PREFENCE/minehotfeed` 仓库了 |
| 20-30 分钟时间 | 新手第一次操作预留 1 小时 |
| 你的域名 `prenceyours.xyz` | 可选，没域名也能跑 |

## 整体流程图

```
注册 Cloudflare → 装 Node.js → 创建 D1 数据库 → 部署 Worker 后端
                                                          ↓
                                             拿到 API 地址 https://xxx.workers.dev
                                                          ↓
                              打开浏览器看效果 ← 部署前端到 Pages / GitHub Pages
```

# 第一部分：注册和准备

## 步骤 1：注册 Cloudflare 账号（5分钟）

### 1.1 打开注册页面
在浏览器地址栏输入：
```
https://dash.cloudflare.com/sign-up
```

### 1.2 填写注册信息
你会看到这个表单：

```
┌──────────────────────────────┐
│ Email:  [你的邮箱            ]│  ← 填你的常用邮箱
│ Password:[设置密码           ]│  ← 至少8位，包含字母+数字
│         [再次输入密码        ]│
│                               │
│  [ □ ] 我同意服务条款         │  ← 勾上
│                               │
│  [    创建账号    ]            │  ← 点击
└──────────────────────────────┘
```

### 1.3 验证邮箱
去你的邮箱，找到 Cloudflare 发的验证邮件，点里面的 **Verify Email** 链接。

> 如果没收到，检查垃圾邮件箱。

### 1.4 登录后看到的界面
验证完成后，浏览器会自动跳转到 Cloudflare 控制面板（Dashboard）。左侧有一排菜单，你会经常用到这些：

```
┌─────────────────────────────┐
│ Cloudflare                  │
├─────────────────────────────┤
│ 🏠 主页                     │
│ 📊 概述                     │
│ 🌐 网站                     │  ← 域名管理（后面用到）
│ ⚡ Workers 和 Pages         │  ← 核心！D1/Worker/Pages 都在这里
│ 💾 D1 SQL 数据库            │  ← 数据库入口
│ 📦 R2 对象存储              │
│ ...                         │
└─────────────────────────────┘
```

**到这里，第 1 步完成 ✅**

## 步骤 2：安装 Node.js（10分钟）

> 如果已经装了 Node.js 18+，跳到步骤 3。

### 2.1 下载安装包
打开 https://nodejs.org

页面中间有两个按钮：
- **LTS（推荐）**—— 点左边的，版本号如 `20.x.x`
- Current —— 不用管

### 2.2 Windows 安装
1. 双击下载的 `.msi` 文件
2. 一路点 **Next**，所有选项保持默认
3. 最后点 **Install**
4. 安装完成后点 **Finish**

### 2.3 Mac 安装
1. 双击下载的 `.pkg` 文件
2. 一路点 **继续**
3. 输入你的 Mac 登录密码
4. 点 **安装软件**

### 2.4 验证安装
打开终端/命令提示符：

**Windows**：按 `Win + R`，输入 `cmd`，回车
**Mac**：按 `Cmd + 空格`，输入 `terminal`，回车

在终端里输入：
```bash
node -v
```

应该看到类似 `v20.11.0` 的输出。如果看到了，说明装好了。

### 2.5 安装 wrangler（Cloudflare 命令行工具）
在同一个终端里输入：
```bash
npm install -g wrangler
```

等它跑完（大约 1 分钟），然后验证：
```bash
wrangler --version
```

应该看到类似 `⛅️ wrangler 3.x.x` 的输出。

### 2.6 登录 Cloudflare
```bash
wrangler login
```

执行后：
1. 浏览器会自动打开一个 Cloudflare 授权页面
2. 点 **Allow / 允许**
3. 看到 "Success" 后，回到终端

终端会显示 `✅ Successfully logged in.`

**到这里，第 2 步完成 ✅**

# 第二部分：创建数据库

## 步骤 3：创建 D1 数据库（5分钟）

### 3.1 进入 D1 创建页面
在 Cloudflare Dashboard 里：
1. 点左侧菜单 **Workers 和 Pages**
2. 在顶部子标签里点 **D1 SQL 数据库**
3. 点蓝色按钮 **创建数据库**

### 3.2 填写信息
```
┌──────────────────────────────────┐
│ 数据库名称：                      │
│ ┌──────────────────────────────┐ │
│ │ hotfeed-db                   │ │  ← 名字就是 hotfeed-db
│ └──────────────────────────────┘ │
│                                  │
│ 位置：                           │
│ ┌──────────────────────────────┐ │
│ │ APAC (亚太地区)       ▾      │ │  ← 选 APAC，国内访问更快
│ └──────────────────────────────┘ │
│                                  │
│        [  创建数据库  ]           │  ← 点击
└──────────────────────────────────┘
```

### 3.3 复制 Database ID
创建完成后进入数据库详情页，页面顶部有：

```
Database ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx   📋
```

**点旁边的 📋 复制按钮，把这个 ID 记下来。** 可以临时粘到记事本里，后面要用。

### 3.4 建表
在数据库详情页，点顶部的 **控制台** 标签，你会看到一个 SQL 编辑器：

```
┌──────────────────────────────────────────────────────┐
│ [控制台]  [表]  [指标]  [设置]                        │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │                                                │  │
│  │  在这里粘贴 SQL...                              │  │
│  │                                                │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│              [  执行查询  ]                           │
└──────────────────────────────────────────────────────┘
```

把下面的 SQL 语句**全部复制**，粘贴到编辑器里，点 **执行查询**：

```sql
CREATE TABLE IF NOT EXISTS dedup_hotspots (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    subtitle TEXT DEFAULT '',
    unified_heat REAL DEFAULT 0,
    category TEXT DEFAULT '社会',
    trend TEXT DEFAULT 'stable',
    platform TEXT DEFAULT '',
    url TEXT DEFAULT '',
    region TEXT NOT NULL DEFAULT 'domestic',
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS history_hotspots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    rank INTEGER NOT NULL,
    title TEXT NOT NULL,
    subtitle TEXT DEFAULT '',
    unified_heat REAL DEFAULT 0,
    category TEXT DEFAULT '社会',
    platform TEXT DEFAULT '',
    url TEXT DEFAULT '',
    region TEXT NOT NULL DEFAULT 'domestic',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS fetch_status (
    key TEXT PRIMARY KEY,
    value TEXT
);

CREATE INDEX IF NOT EXISTS idx_dedup_region ON dedup_hotspots(region);
CREATE INDEX IF NOT EXISTS idx_dedup_category ON dedup_hotspots(category);
CREATE INDEX IF NOT EXISTS idx_dedup_updated ON dedup_hotspots(updated_at);
CREATE INDEX IF NOT EXISTS idx_history_date ON history_hotspots(date);
```

执行成功会显示绿色的 "✅ Success"。点 **表** 标签，应该能看到 3 张表：`dedup_hotspots`、`history_hotspots`、`fetch_status`。

**到这里，第 3 步完成 ✅**

# 第三部分：部署后端 Worker

## 步骤 4：下载项目代码（5分钟）

### 4.1 克隆仓库
打开终端，输入：
```bash
git clone https://github.com/S-PREFENCE/minehotfeed.git
cd minehotfeed/worker
```

### 4.2 修改配置文件
用任意文本编辑器（记事本就行）打开 `wrangler.toml`，找到这一段：

```toml
[[d1_databases]]
binding = "DB"
database_name = "hotfeed-db"
database_id = "你的D1数据库ID"   # ← 把这里替换成步骤 3.3 复制的真实 ID
```

替换后保存。比如：
```toml
database_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
```

### 4.3 安装依赖
在终端里（确保在 `worker` 目录下）：
```bash
npm install
```

等 1-2 分钟，看到 `added xxx packages` 就完成了。

**到这里，第 4 步完成 ✅**

---

## 步骤 5：部署到 Cloudflare（2分钟）

### 5.1 执行部署命令
在终端里（确保在 `worker` 目录下）：
```bash
npx wrangler deploy
```

你会看到类似这样的输出：
```
 ⛅️ wrangler 3.x.x
─────────────────────────────────────────────
Total Upload: 15.42 KiB / gzip: 4.28 KiB
Worker Startup Time: 5 ms
Your worker has access to the following bindings:
- D1 Databases:
  - DB: hotfeed-db (xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)
Uploaded hotfeed-backend (2.45 sec)
Deployed hotfeed-backend triggers
  https://hotfeed-backend.你的用户名.workers.dev
  schedule: */10 * * * *
Current Deployment ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### 5.2 记下你的 API 地址
最重要的就是这一行：
```
https://hotfeed-backend.你的用户名.workers.dev
```

**把这个 URL 复制下来，保存到记事本！** 这是你的后端 API 地址，后面前端要用。

> 📝 你的 URL 中 `你的用户名` 那部分是你 Cloudflare 账号自动分配的子域名，每个人不一样。

**到这里，第 5 步完成 ✅**

---

## 步骤 6：验证后端是否正常（3分钟）

### 6.1 健康检查
在浏览器地址栏输入（把 `<你的API地址>` 换成步骤 5.2 记下的地址）：
```
<你的API地址>/api/v1/health
```

例如：
```
https://hotfeed-backend.zhuanz.workers.dev/api/v1/health
```

**应该看到：**
```json
{"status":"ok","timestamp":"2025-01-15T10:30:00.000Z"}
```

### 6.2 手动触发首次抓取
在同一个浏览器里，按 F12 打开开发者工具，点 **Console（控制台）** 标签，粘贴这段代码回车：

```javascript
fetch('<你的API地址>/api/v1/refresh', { method: 'POST' })
  .then(r => r.json())
  .then(data => console.log('结果:', data))
  .catch(err => console.error('失败:', err))
```

> ⚠️ 首次抓取需要 30-60 秒（要同时抓 18 个平台）。控制台会一直显示 pending，正常。

**成功返回示例：**
```json
{"ok":true,"domestic":45,"international":30,"failed":["微博","Reddit"]}
```

`failed` 里有几个失败平台是正常的，免费 API 偶尔抽风。只要 `domestic` 和 `international` 数量 > 0 就行。

### 6.3 查看数据
浏览器访问：
```
<你的API地址>/api/v1/hotspots?region=all&limit=10
```

应该看到国内和国际各 10 条热点数据。

**到这里，第 6 步完成 ✅**

# 第四部分：部署前端

## 步骤 7：配置前端 API 地址（2分钟）

### 7.1 修改配置文件
用文本编辑器打开项目中的 `src/frontend/src/composables/useHotspots.js`，找到第 6 行附近：

```javascript
const API_BASE = 'https://hotfeed-backend.你的用户名.workers.dev/api'
```

把 URL 换成你在步骤 5.2 记下的真实地址，**保留末尾的 `/api`**。

例如：
```javascript
const API_BASE = 'https://hotfeed-backend.zhuanz.workers.dev/api'
```

保存文件。

### 7.2 构建前端
在终端里：
```bash
cd src/frontend
npm install
npm run build
```

构建完成后，`dist` 目录下就是可以部署的静态文件。

**到这里，第 7 步完成 ✅**

# 第五部分：绑定自定义域名（可选）

## 步骤 9：绑定 prenceyours.xyz

> 如果你的域名 DNS 已经托管在 Cloudflare，这一步 2 分钟搞定。
> 如果域名在别处买的，先做步骤 9.1，再做 9.2。

### 9.1 域名接入 Cloudflare（仅域名不在 Cloudflare 时做）

1. Dashboard 左侧点 **网站** → **添加站点**
2. 输入 `prenceyours.xyz` → 点 **继续**
3. 选择 **Free** 计划 → 点 **继续**
4. Cloudflare 会扫描现有 DNS 记录 → 直接点 **继续**
5. 你会看到两个 **Cloudflare 名称服务器**，类似：
   ```
   eve.ns.cloudflare.com
   hank.ns.cloudflare.com
   ```
6. **复制这两个 NS 地址**
7. 打开你的域名注册商网站（买域名的那个网站）
8. 找到 DNS/NS 设置 → 把原来的 NS 替换为 Cloudflare 的两个
9. 等待生效（几分钟到几小时，通常很快）

### 9.2 Pages 绑定域名

1. Dashboard → **Workers 和 Pages** → 点你的 `hotfeed-frontend` 项目
2. 顶部点 **自定义域** 标签
3. 点 **设置自定义域**
4. 输入 `prenceyours.xyz`
5. 点 **继续** → 点 **激活域**
6. 等待 DNS 和 SSL 自动配置完成（1-2 分钟）

### 9.3 访问测试
浏览器打开 `https://prenceyours.xyz`，应该看到和 Pages 地址完全一样的页面。

# 常见问题排查

## ❌ 终端提示 "wrangler: command not found"
**原因**：wrangler 没装上或 PATH 没配好。

**解决**：
```bash
npm install -g wrangler
# 如果还不行，关掉终端重新打开试试
```

## ❌ 部署时报 "Could not find wrangler.toml"
**原因**：你不在 `worker` 目录下。

**解决**：
```bash
cd minehotfeed/worker
npx wrangler deploy
```

## ❌ 浏览器访问 API 返回 404
**检查清单**：
1. URL 拼写是否正确（`/api/v1/health` 前面没有多余空格）
2. 是不是访问的 **https** 不是 http
3. Worker 是否真的部署成功了（去 Dashboard → Workers 和 Pages 确认）

## ❌ /api/v1/refresh 返回空的 domestic 和 international
**原因**：免费数据源偶尔不稳定，属于正常现象。

**解决**：
1. 等 10 分钟让 Cron 自动重试
2. 或者再手动 POST 一次 `/api/v1/refresh`
3. 查看 Worker 日志：
   ```bash
   cd minehotfeed/worker
   npx wrangler tail
   ```

## ❌ 前端页面能打开但数据加载不出来
**检查清单**：
1. 确认 `useHotspots.js` 里的 `API_BASE` 已经改成了你的真实 Worker 地址
2. 确认地址末尾没有多余的 `/`（应该是 `/api` 不是 `/api/`）
3. 打开浏览器 F12 → Console 看有没有红色报错
4. 如果报 CORS 错误，说明 API_BASE 配错了

## ❌ 域名绑定后访问不了
**检查清单**：
1. 等 5 分钟再试（DNS 传播需要时间）
2. 确认域名 NS 已经改到了 Cloudflare
3. 在 Cloudflare → 网站 → DNS 里确认有对应的记录

## ❌ 想修改 Worker 代码后重新部署
```bash
cd minehotfeed/worker
# 改完代码后
npx wrangler deploy
```

## ❌ 想把前端改回 GitHub Pages
你的 GitHub Pages 已经在运行了，地址是：
```
https://s-prefence.github.io/minehotfeed/
```
只需确保 `useHotspots.js` 里的 `API_BASE` 指向正确的 Worker 地址，然后：
```bash
cd src/frontend
npm run build
# 把 dist 内容推到 gh-pages 分支
```
