# HotFeed v4 混合架构 — 一键部署手册

> **最后更新**：2026-07-12  
> **预计耗时**：5 分钟（一次性），之后全自动运行

---

## 架构说明

```
[你的电脑] → Python脚本定时抓取UAPI → POST推送到Worker → Worker写入D1
                                                              ↓
[用户浏览器] ← GitHub Pages前端 ← GET /api/v1/hotspots ← Worker从D1读取
```

**为什么这样？** Cloudflare Workers 的 IP 被国内 API 屏蔽，无法直接抓取。但你的本地电脑可以！所以改成"本地抓取 → 推送到 Worker"的混合模式。

---

## 第 1 步：更新 Worker 代码（仅此一次手动操作）

1. 打开 https://dash.cloudflare.com/
2. 左侧菜单 → **Workers & Pages**
3. 点击 **hotfeed-backend**
4. 点击右上角 **编辑代码**（Edit Code）
5. **全选删除**所有旧代码（Ctrl+A → Delete）
6. 打开下方文件，**全选复制**全部内容：

   📄 **`worker/deploy-v4.js`**（在项目目录中）

7. 粘贴到编辑器中
8. 点击右上角 **保存并部署**（Save and Deploy）
9. 验证：浏览器访问 `https://hotfeed-backend.2628944969.workers.dev/api/v1/health`
   → 应返回 `{"status":"ok","timestamp":"...","mode":"hybrid"}`

---

## 第 2 步：启动本地抓取守护进程（全自动）

### 前置条件
- Python 3.8+（检查：`python --version`）
- 安装依赖：`pip install requests`

### 启动方式

**方式 A：一键启动（推荐）**
```
双击 local-fetcher/start.bat
```

**方式 B：命令行启动**
```cmd
cd local-fetcher
python fetch_and_push.py --daemon
```

**方式 C：单次测试**
```cmd
cd local-fetcher
python fetch_and_push.py --dry-run    # 只抓取不推送（测试）
python fetch_and_push.py              # 抓取并推送一次
```

### 守护进程说明
- 每 10 分钟自动抓取所有国内平台热点
- 自动推送到 Cloudflare Worker
- Worker 自动存入 D1 数据库
- 前端自动展示（前端每 10 分钟轮询刷新）
- 窗口可最小化到托盘，不要关闭

---

## 第 3 步：验证效果

1. 打开前端：https://s-prefence.github.io/minehotfeed/
2. 强制刷新：**Ctrl+Shift+R**
3. 应该看到国内热点卡片（微博、抖音、百度、知乎、头条、B站等）
4. 国际热点由 Worker Cron 自行抓取（Reddit、Google News、BBC、CNN）

---

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端显示空白 | 缓存旧版 | Ctrl+Shift+R 强制刷新 |
| 前端显示"暂无数据" | Worker 还没收到推送 | 等 Python 脚本运行一次（首次约30秒） |
| Python 脚本报 429 | UAPI 限流 | 脚本已内置延迟，等待下次自动重试 |
| 国际热点为空 | 国际源由 Worker Cron 抓取 | 等最多10分钟，Worker Cron 会自动执行 |
| 守护进程窗口关了 | 进程终止 | 重新双击 start.bat |

## 项目文件说明

```
HotFeed聚合热点/
├── local-fetcher/          ← 本地抓取脚本（在你自己电脑上运行）
│   ├── fetch_and_push.py   ← Python 抓取+推送脚本
│   └── start.bat           ← Windows 一键启动
├── worker/
│   ├── deploy-v4.js        ← Worker 代码（粘贴到 Cloudflare Dashboard）
│   └── wrangler.toml       ← Wrangler 配置（参考用）
├── src/frontend/           ← Vue 前端源码
└── docs/                   ← 文档
```
